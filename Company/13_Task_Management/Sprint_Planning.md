# Sprint Planning
## Sprint Structure
- **Duration:** 2 weeks (configurable, start with 2 weeks for Tiny World).
- **Capacity:** Based on founder availability (20-25 productive hours/week) and agent throughput.
- **Cadence:** Sprint Planning → Execution → Sprint Review → Sprint Retro.
## Sprint Planning Process
1. **Backlog Grooming** (Product Manager Agent, Day -1): Review and prioritize backlog items for next sprint.
2. **Capacity Check** (Founder Agent): Confirm founder availability and planned review blocks.
3. **Sprint Goal** (Product Manager Agent + Founder): Define 1-2 sprint goals aligned with active epic milestones.
4. **Task Selection** (Product Manager Agent): Select tasks from READY state that fit capacity and serve sprint goals.
5. **Agent Assignment** (per Assignment Rules 13-007): Assign tasks to appropriate agents.
6. **Sprint Start** (All agents): Tasks move from READY → IN_PROGRESS.
## Sprint Review (End of Sprint)
- What was completed vs. planned
- Demo of working features
- KPI impact of completed work
- Carry-over items and reasons
## Sprint Metrics
| Metric | Target |
|--------|--------|
| Sprint completion rate | >80% of planned tasks DONE |
| Carry-over rate | <20% of tasks carried to next sprint |
| Sprint goal achievement | 100% of sprint goals met |
