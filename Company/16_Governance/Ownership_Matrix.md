# Ownership Matrix
## 1. Vision
Every document, workflow, subsystem, and game feature must have a single defined owner. Shared ownership is no ownership.

## 6. Requirements
### 6.1 Ownership Rules
1. **The Owner maintains the asset.** They are responsible for its freshness, quality, and updates.
2. **The Owner approves changes.** Even if another agent writes a draft, the Owner must review and approve it.
3. **If the Owner is retired**, their assets must be explicitly reassigned before retirement (per `08-003`).

### 6.2 Core Ownership Mapping
| Asset Category | Owner Agent |
|----------------|-------------|
| Documentation System | Documentation Agent |
| Quality Pipeline | QA Agent / Code Review Agent |
| Agent OS Infrastructure | CTO Agent (or Automation Agent) |
| Product Backlog | Product Manager Agent |
| Analytics Infrastructure | Analytics Agent |
| Core Engine Architecture | Unity Architect Agent |
| Monetization Strategy | Monetization Agent |
